8.高效编程

概念 描述 例子
Big O（大 O） 描述算法在输入规模增大时，最坏情况下的运行时间或空间复杂度的上限。 线性搜索：O(N)
Big Omega（大 Ω） 描述算法在最好的情况下，运行时间或空间复杂度的下限。 线性搜索的最佳情况：Ω(1)
Big Theta（大 Θ） 描述算法运行时间或空间复杂度的精确边界（既是上限也是下限）。 线性搜索的总体情况：Θ(N)
Big O 并不表示最坏情况 虽然常用于最坏情况，但 Big O 只表示上限，仍然可以使用 Big Theta 来描述最坏情况。 使用 Big Theta 描述最坏情况
Big Omega 并不表示最佳情况 Big Omega 表示下限，并不专指最佳情况，仍然可以使用 Big Theta 来描述最佳情况。 使用 Big Theta 描述最佳情况
摊销分析 一种用于证明一系列操作的平均操作成本的分析方法。 动态数组插入的平均时间复杂度 O(1)

9.不相交集

如果两个集合没有共同元素，则它们被称为不相交集合。不相交集合（或并查集）数据结构跟踪划分为多个不相交集合的固定数量的元素。该数据结构有两种操作：

connect(x, y)：连接x和y。也称为union
isConnected(x, y)：如果x和y是连通的（即属于同一集合），则返回 true。

不相交集设计的四次迭代：快速查找 → 快速合并 → 加权快速合并 (WQU) → 带路径压缩的 WQU。
使用数组实现，数组的坐标代表值

快速查找：将同一类的都给相同的值。
快速合并：将坐标的值设为它的上一个节点，根节点设为-1
加权快速合并：合并的时候将小的并到大的上面去，根节点储存size
路径压缩：将使用函数find的时候经过的节点都绑到根节点上面

Implementation isConnected connect
Quick Find Θ(N) Θ(1)
Quick Union O(N) O(N)
Weighted Quick Union (WQU) O(log N) O(log N)
WQU with Path Compression O(α(N))* O(α(N))*

10.ADT

抽象数据类型（ADT）是根据操作而不是实现来定义的。

几个有用的ADT：

  • 不相交集，映射，集合，列表。
  • Java 提供了 Map、Set、List 接口以及一些实现。

我们已经看到了两种实现 Set（或 Map）的方法：

  • ArraySet:o(N) operations in the worst case.
  • BST:o(logN) operations if tree is balanced.

BST 实现：

  • 搜索和插入很简单（但插入有点棘手）。
  • 删除更具挑战性。典型的方法是“Hibbard 删除”。

BST

可以将 BST 视作是有序的二叉树，这在搜索的时候非常方便与迅速

类比二分查找，当我们想要查找 BST 中的数据的时候，思路是同样的：

和当前结点的值比大小，若是则返回
若是小于待查值，则往右拐
若是大于待查值，则往左拐
速度贼快！如果树很茂密，那么树的高度 H = log N，即意味着查找用时也是 log N

添加的思路是同样的

删除操作则需要分类讨论：

  • 被删除结点无子节点
  • 被删除结点有一个子节点
  • 被删除结点有两个子节点

若无子节点，直接删除即可

若有一个子节点，子承父业

若有两个子节点，即选择一个新节点替换之，另外一个就添加到新结点的下面。

12.哈希表

经过我们从 BST-BT-LLRB 的不断优化，查询的运行用时已经被我们做到了 log N 的境界，那么，还能进一步变强吗？(或者以空间换时间？)

基本思路是直接使用数组来替代上面那么复杂的数据结构——以空间换时间！直接用数组的 index 代表数据，用 true/false 代表是否被存储——

如何适用除数字以外的其他数据？——使用 ASCII！或者 Unicode！
不可避免的冲突？——内部使用 List 结构！
内存开销巨大？运行时不稳定？——使用限定大小且动态增长的数组！同时进一步改进我们的 Hashcode!,考虑负载率也就是元素除以使用的下标。

哈希码具有三个必要属性，

  • 它必须是整数
  • 如果我们.hashCode()对一个对象运行两次，它应该返回相同的数字
  • 考虑的两个对象.equal()必须具有相同的哈希码。
  • 另外哈希应该尽量均匀分布

13.堆和优先级队列

我们使用堆来实现优先级队列，快速找到最小值或者最大值。
堆是改良的二叉搜索树。
我们将二叉最小堆定义为完整的并遵循最小堆属性：

  • 最小堆：每个节点都小于或等于其两个子节点
  • 完整：仅底层缺少项目（如果有），所有节点都尽可能位于左边。

我们用堆要实现PQadt的三个方法。

  • add：暂时添加到堆尾。沿层次结构向上移动到适当位置。
    如果子节点 < 父节点，则下浮涉及交换节点
  • getSmallest：返回堆的根这保证了我们的最小堆属性是最小的
  • removeSmallest：将堆中的最后一个项交换到根。将层次结构下沉到适当的位置。如果父节点 > 子节点，则下沉涉及交换节点。与最小的子节点交换以保留最小堆属性。

14.简要总结

到目前为止，我们讨论的所有数据结构都是为了解决搜索问题。我们学过的每个数据结构都用于以在特定场景中提高搜索效率的方案存储信息。

Name Store Operation(s) Primary Retrieval Operation Retrieve By
List add(key), insert(key, index) get(index) index
Map put(key, value) get(key) key identity
Set add(key) containsKey(key) key identity
PQ add(key) getSmallest() key order (aka key size)
Disjoint Sets connect(int1, int2) isConnected(int1, int2) two integer values

这些是抽象数据类型。这意味着我们定义行为，而不是实现。我们也可以有很多种实现方法。
