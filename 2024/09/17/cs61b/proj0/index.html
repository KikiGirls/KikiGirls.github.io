Author: 王思懿 Date: September 17, 2024  21:20:28 Category: Cs61b

cs61B 的第一个项目就是实现 2048 这个小游戏。题目不难，英文不怎么好理解比较困难。

emptySpaceExists

判断这个游戏里面有没有空的地方。有就返回ture

通过观察代码发现调用 size() 来得到板子的边界条件，然后如果是空的块的话，tile() 会为空。

于是我们就可以遍历每个格子然后判断

maxTileExists

判断是否有格子的值达到最高，但我们要注意空格子是没有值的，而不是值为0，所以在遍历的时候要先判断是不是空格子。

atLeastOneMoveExists

判断是否能够移动
条件1，有空格子。条件二，相邻格子有相同的值。

对条件1，直接使用上一个的函数。

对于条件2，我们可以让最后一列以外的所有格子检查右边的格子是否有一样的值，这样就完成了横向的检查，纵向的就让除最后一行以外的所有格子检查自己下面的。

Building the Game Logic

tilt

这里要做的就是当我们输入方向键的时候，所有格子要向同一个方向移动，然后有相同的要合并，但是只能合并一次，比如224，移动后只能变为44，而不是8。同时如果这些格子移动了，我们要让变量chenged变成true。

实际上我们只需处向上的移动，如果我们收到向下指令，我们可以将棋盘倒转，然后再向上移动，最后再倒转回去，在这里已经为我们写好了这个函数。

现在我们只需考虑向上的情况了，那么实际上我们也只需要考虑同一列上的移动，不同的列是不影响的。我们对不同的列做一个遍历就可以了

在向上移动的时候，最上方的格子实际上是不动的，那么我们只需要遍历最上方以后的格子，但由于最下方的格子为0，所以我们就要倒序遍历。

for (int row = size() - 2; row >= 0; row--)

对于某一个具体的格子我们有几种情况，

  • 它前面有空格，可以往前走
  • 它前面有和它值不一样的格子。
  • 它前面有和它值一样的格子，可以合并。
  • 它前面有和它值一样的格子，但是那个格子已经合并过了。所以只能走到下面的空格子。

那么我们就要首先判断它前面有几个空格子。它需要往前面走多远。当我们碰到墙壁或者其他有值的格子时候就停止计数

当我们遇到1.2情况的时候我们就可以前进，但是当是4情况的时候怎么办呢，我们就需要判定前面的合并过没，我们引入一个变量，最开始的时候为true，合并过了就变为F,只移动没合并就变为T.这样就可以通过判断这个变量来确定合并过了就不再合并了。
boolean Merge; Merge = true;

这个时候剩下的就是情况2了
