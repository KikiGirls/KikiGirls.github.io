这个还是花了我不少时间，感觉自己效率不怎么高，有很多格式不理解导致了效率很慢。

在这个项目里面我们将实现双端队列，注意遵守风格指南。

lldDeque

用链表的结构实现双边队列。要实现链表的话，我们就要先实现，链表的节点，有点类似于火车车厢，每个车厢是独立却又连接到一起的。车厢有前后两个通道，5号车厢往前走，就是4号，往后走就是6号。可是我们如何分别哪节是第一节呢，于是我们就需要一个没有值的节点来作为我们的车头，第0号车厢。为了实现双端，我们有两种方法，一就是再规定一个车尾，二就是把车厢做成一个环。第0号车厢的后面是第一节，前面是最后一节。官方文档建议用循环的方法。

Node

首先我们先来实现，这个火车的基本元素，车厢，也就是节点。

classMethods

构造函数

首先有两个基本属性,长度和用来标记车头和车尾的哨兵节点。接着实现构造函数。

public int size()

public void addLast(T item)

在队列的最后面添加元素，让原来最后一个节点的后面，为要添加的节点，由于添加的节点成为了最后一个，那么哨兵节点的前面一个就变成了添加的这个。

public void addFirst(T item)

同理。

public T removeFirst()

移除第一个，将原来第一个后面一个，也就是第二个，和哨兵节点连接，这样第二个就变成第一个了。

public T removeLast()

同理

public T get(int index)

通过给定的坐标返回相应坐标的值。从第一个节点往后数相应的数就行。

public T getRecursive(int index)

adDeque

利用数组来定义，我的方法比较麻烦，
项目原话是这样的：
您需要以某种方式跟踪哪些数组索引保存了 Deque 的前端和后端元素。我们强烈建议您将数组视为循环数组，以进行此练习。换句话说，如果您的前端项位于位置零，并且您 addFirst，则新的前端应该循环回到数组的末尾（因此，双端队列中的新前端项将是底层数组中的最后一个项）。这将比非循环方法少很多麻烦。有关更多详细信息，请参阅项目1 演示幻灯片 。
