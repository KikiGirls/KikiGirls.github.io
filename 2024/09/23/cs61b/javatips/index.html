语法之类的

Author: 王思懿 Date: September 23, 2024  18:19:28 Category: Cs61b

  • Java 自带的 Cloneable 接口可以用于对象的浅拷贝。若要实现深拷贝，需在 clone() 方法中手动拷贝对象的内部可变引用类型字段。

  • 驼峰命名（Camel Case）是一种常见的命名规范，通常用于编程中定义变量、函数或类名。其特点是：
    • 第一个单词以小写字母开头。
    • 后续的每个单词首字母大写。
    • 不使用空格或下划线来分隔单词。

  • 这个错误是因为在类的某个作用域中，局部变量或方法参数的名称与类中的字段名称相同，从而隐藏了类字段。为了避免这个问题，你需要使用不同的名字来区分局部变量或方法参数与类字段，或者可以使用 this 关键字来明确指定你访问的是类的字段。

解决方案：
1. 使用不同的名称：你可以将方法参数或局部变量的名称改为不与类字段冲突的名字。
2. 使用 this 关键字：如果你想继续使用相同的名字，可以在类字段前加上 this 来明确表示你指的是类的字段，而不是方法参数或局部变量

  • 在你的 equals 方法中有一个问题，变量 other 在两个 if 语句块中定义了两次，因此在块之外是不可见的。你需要确保 other 是在 if-else 结构之外定义的，这样在后续代码中能够访问到。另外，需要将 other 定义为 Deque<?> 类型，因为你想要比较两个实现 Deque 接口的类。

  • 如果 if-else 语句很简单，你可以使用三元运算符来简化代码：
    int a = 5;
    String result = (a > 3) ? “a 大于 3” : “a 小于或等于 3”;
    System.out.println(result);

    1. 静态方法
      静态方法属于类本身，而不是类的某个实例，可以通过类名直接调用。
      public static int square(int x) {
      return x * x;
      }
      int result = MyClass.square(5); // result 的值为 25

  • break 语句可以用于提前终止循环，即使循环条件仍然为 true。

    1. while 循环结合 continue

continue 语句用于跳过当前循环的剩余部分，并继续下一次迭代。

    1. do-while 循环

do-while 是另一种循环，它至少会执行一次，即使条件为 false。它的语法如下：

do {
// 循环体
} while (条件);

即使条件一开始为 false，但 do-while 循环确保了循环体至少会执行一次。输出结果：
int i = 10;
do {
System.out.println(“i is: “ + i);
i++;
} while (i < 5);

  • 在 Java 中，不能动态地在方法内给对象添加新属性。你可以通过以下方式实现类似的效果：

    • 在类中定义属性并使用方法修改它们。
    • 使用 Map 结构存储动态属性。
    • 使用继承扩展类以增加新的属性。
    • 使用组合模式在其他类中添加额外属性。

  • 在 Java 中，局部变量无法在方法之间直接传递和修改。但可以通过以下几种方法来实现相同的效果：

    • 使用类的成员变量：让方法共享同一个实例变量，直接修改。
    • 通过参数传递对象的引用：通过传递引用类型对象的参数，在方法中修改对象的属性。
    • 使用返回值：通过方法返回修改后的值，并用这个值更新原变量。
    • 使用数组或集合：数组和集合是引用类型，传递后可以修改它们的内容。

  • •	||：表示 “或”，只要其中一个条件为 true，整个条件为 true。

    • &&：表示 “与”，所有条件为 true，整个条件才为 true。
    • !：表示 “非”，将条件的真假值取反。

  • for (int item : deque) {
    System.out.println(item); // 输出: 1 2 3
    }
    通过实现 Iterable 接口并提供 iterator() 方法，你的 LinkedListDeque 对象就可以支持增强的 for-each 循环，或其他基于 Iterator 的操作。
    Iterator 是 Java 中的一个接口，用于遍历集合中的元素（如 List、Set、Deque 等）。它提供了一种标准化的方式来访问集合中的元素，而无需了解底层数据结构的实现细节。

    • 遍历集合：使用 Iterator，你可以顺序访问集合中的每一个元素，而不需要关心集合是链表、数组还是其他类型的结构。
    • 提高代码可读性：通过迭代器，代码可以更加简洁和可维护。
    • 安全删除元素：Iterator 允许在遍历过程中安全地删除集合中的元素（使用 remove() 方法）。

  • •	System.out.println()：自动换行，适合简单的输出。

    • System.out.print()：不换行，适合拼接连续输出。
    • System.out.printf()：用于格式化输出，可以控制输出的格式、精度和对齐。
