Style Guide

Author: 王思懿 Date: September 16, 2024  0:20:28 Category: Cs61b

风格指南
作者：姚艾伦
关于本风格指南的说明：通过运行自动风格检查器可能更容易了解这些规则。（以后将提供有关如何运行自动风格检查器的更多说明。）

空格
每个文件必须以换行符序列结尾。

文件不得包含水平制表符。仅使用空格进行缩进。

任何行都不能包含尾随空格。

不要添加空格：
通用类型标识中的“<”和“>”周围（“List”，而不是“列表”，或 “List“。
前缀运算符“！”，“–”，“++”，一元运算符“-”或一元运算符“+”之后。
在标记“；”或后缀运算符“–”和“++”之前。
在“(”之后或“)”之前。
后 ”。”
当你孤独的时候。
请添加空格：
在“;”、“,”或类型转换之后（例如，“(String) x”，而不是“(String)x”）。
围绕二元运算符（例如“*”、“+”）和比较运算符。
围绕赋值运算符（例如“=”、“+=”）。
三元条件运算符（“x>0 ? x : -x”）中的“？”和“：”周围。
围绕关键字“assert”、“catch”、“do”、“else”、“finally”、“for”、“if”、“return”、“synchronized”、“try”、“while”。
一般来说，在操作符前换行（插入换行符），例如

… + 20 * X
+ Y;
不要用空格将方法调用中的方法名称与“（”分隔开。但是，在长行中，你可以用换行符和空格（用于缩进）将它们分隔开。
缩进
基本缩进步骤是 4 个空格。

按每个块级别的基本缩进步骤缩进代码（块通常用“{”和“}”括起来），例如

if (x > 0) {
r = -x;
} else {
r = x;
}
缩进“case”标记超出其封闭“switch”的缩进，例如

switch (op) {
case ‘+’:
addOpnds(x, y);
break;
default:
ERROR();
}
通过基本缩进步骤缩进连续行。

牙套
使用 { } 括号括起所有“if”、“while”、“do”和“for”语句。

将“}”括号放在“else”、“finally”或“catch”的同一行，例如

if (x > 0) {
y = -x;
} else {
y = x;
}
将打开块的“{”放在行末。通常，它位于包含它的“if”、“for”、“while”、“switch”、“do”、方法头或类头的末尾。如果行长迫使它移到下一行，请不要缩进，而是将其单独放在一行上。

评论
方法应该有 javadoc 注释来解释行为、参数（使用 @param 标签或其他方式）和返回类型。

返回非 void 值的方法必须在其 Javadoc 注释中使用“@return”标记或包含单词“return”、“returning”或“returns”的运行文本中的短语来描述它们。

每个 Javadoc 注释都必须以正确格式的句子开头，以大写字母开头，以句号结尾。

名字
静态最终常量的名称必须全部大写（例如，RED、DEFAULT_NAME）。

参数、局部变量和方法的名称必须以小写字母开头，或者由单个大写字母组成。

类型（类）的名称，包括类型参数，必须以大写字母开头。

包的名称必须以小写字母开头。

实例变量和非最终类（静态）变量的名称必须以小写字母或“_”开头。

导入
不要使用“import PACKAGE。”，除非包是 java.lang.Math、java.lang.Double 或 org.junit.Assert。“import static CLASS。 ”就可以了。

不要两次导入同一个类或静态成员。

不要导入您不使用的类或成员。

各种 Java 样式约定
数组类型应在元素类型名称后加上“[]”，而不是在声明符后。应写“String[] names”，而不是“String names[]”。

按以下顺序编写方法、类或字段的任何修饰符：
公共，受保护或私人。
抽象或静态的。
最终的、暂时的、或易变的。
同步。
本国的。
strictfp。
不要显式地修改多余的方法、字段或类：
不要将接口或注释中的方法标记为“公共”或“抽象”。
不要将接口或注释中的字段标记为“静态”，“公共”或“最终”。
不要将 final 类中的方法标记为“final”。
不要将嵌套接口标记为“静态”。
不要将空块（‘{ }’，其中只有空格或注释）用于控制语句。有一个例外：catch 块可以仅由具有以下形式的注释组成

/* Ignore EXCEPTIONNAME. */
避免在代码中使用“魔法数字”，只需为其赋予符号名称即可，例如

public static final MAX_SIZE = 100;
例外是数字 -10 到 10、0.5、-0.5、0.25、-0.25。

不要尝试捕获异常 Exception、RuntimeError 或 Error。

写“b”而不是“b == true”，写“！b”而不是“b == false”。

代替

if (condition) {
return true;
} else {
return false;
}
只需

return condition;
只有类的静态最终字段可以是公共的。其他字段必须是私有的或受保护的。

仅具有静态方法和字段的类不能具有公共（或默认）构造函数。

仅具有私有构造函数的类必须声明为“final”。
避免容易出错的结构
如果一个类覆盖了“equals”，那么它也必须覆盖“hashCode”。

局部变量和参数不得遮盖字段名称。处理仅控制字段的 getter/setter 方法的首选方法是在字段名称前加上“_”，例如

public double getWidth() {
return _width;
}

public void setWidth(double width) {
_width = width;
}
不要使用嵌套赋值，例如“if ((x = next()) != null) …”。虽然这在 C 中很有用，但在 Java 中几乎没有必要。

在每个“switch”语句中包含一个“default”情况。

使用“break”语句或以下形式的注释来结束“switch”语句的每个分支

/* fall through */
不要用“==”比较字符串文字。写

if (x.equals(“something”))
而不是

if (x == “something”)
在某些情况下您确实想要使用“==”，但您不太可能在本课中遇到它们。

限制
文件长度不得超过 2000 行。

每行不得超过 100 个字符。

任何方法都不得超过 80 行。

任何方法的参数不得超过 8 个。

每个文件必须只包含一个外部类（嵌套类也可以）。
