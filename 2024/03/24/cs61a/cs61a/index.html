  • 列表推导式是一种简洁的语法，用于从一个或多个可迭代对象创建新的列表。它由一对方括号 [] 组成，在方括号内部包含一个表达式，后跟一个 for 循环或多个 for 循环，用于迭代可迭代对象，并生成新的列表元素。
    以下是列表推导式的一般形式：
    [expression for item in iterable]
    其中：
    expression 是一个表达式，用于对可迭代对象中的每个元素进行操作，生成新的列表元素。
    item 是一个变量，表示可迭代对象中的每个元素。
    iterable 是一个可迭代对象，如列表、元组、集合等。
    列表推导式还可以包含条件语句，以过滤或转换可迭代对象中的元素。例如：
    [expression for item in iterable if condition]
    在这种情况下，只有满足条件的元素才会被包含在生成的列表中。
    以下是一些列表推导式的示例：
    创建一个包含 1 到 5 的平方的列表
    squares = [x**2 for x in range(1, 6)]
    创建一个包含字符串长度大于 3 的元素的列表
    words = [“apple”, “banana”, “orange”, “pear”, “grape”]
    long_words = [word for word in words if len(word) > 3]
    创建一个包含字母 ‘a’ 的元素的列表
    letters = [‘a’, ‘b’, ‘c’, ‘d’, ‘e’]
    a_letters = [letter for letter in letters if letter == ‘a’]
    以上示例展示了列表推导式的基本用法，你可以根据需要自由组合列表推导式的表达式和条件来生成新的列表。

  • 在递归函数中，如果我们在处理子节点的递归调用时加上了 return 语句，那么在处理完一个子节点后就会立即返回，导致无法继续处理其他子节点。这会导致只有一个子节点被处理，而其他子节点被忽略。
    考虑到递归的特性，我们希望递归调用能够遍历整个树的所有节点，而不仅仅是当前节点的一个子节点。因此，在递归函数中，我们通常不在处理子节点的递归调用处加上 return 语句，这样递归调用就会继续执行，直到所有子节点都被处理完毕，然后再返回到上一级调用处。

生成器对象并不直接返回一个列表，而是可以用于生成一个列表。生成器对象本身是一个可迭代对象
直到遇到 yield 关键字产生一个值。然后，生成器函数会暂停执行，并将产生的值返回给调用方。在下一次迭代时，生成器函数会从上次暂停的地方继续执生成器函数使用 yield 关键字来产生值，而不是使用 return 返回值。生成器函数可以用来逐个产生序列中的元素，而不需要一次性将所有元素存储在内存中。行，直到再次遇到 yield 关键字。

在Python中，[:]和.copy()都可以用于创建一个列表的副本，但它们之间存在一些区别：

切片操作 [:] 是对列表进行的一个切片，相当于提取列表的一个子序列。通过[:]可以获得列表的一个浅拷贝，即列表中的元素对象的引用被复制，但列表中的元素对象本身不会被复制。这意味着，如果列表中的元素是可变对象，对副本的修改会影响原始列表，但如果列表中的元素是不可变对象，则不会影响原始列表。
.copy() 方法是列表对象的一个方法，用于创建列表的一个浅拷贝，与切片操作类似，也是复制了列表中元素的引用。使用.copy()方法更直观，但需要调用方法，相对于切片操作来说，可能会稍微慢一些
