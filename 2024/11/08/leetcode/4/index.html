C++ char 类型

char 类型是 C++ 中用于表示单个字符的基本数据类型。以下是 char 类型的一些关键点：

1. 大小

  • char 通常是 1 字节（8 位），表示一个 ASCII 字符。
  • char 的范围通常在 -128 到 127（有符号）或 0 到 255（无符号）。

2. 有符号与无符号

  • char 类型可以是有符号或无符号的，但具体默认情况取决于编译器。
  • 使用 signed char 或 unsigned char 可以明确指定。

3. 字符表示

  • char 常量可以用单引号表示，例如 'A' 表示字符 A。
  • 特殊字符可以用转义字符表示，例如 '\n'（换行）、'\t'（制表符）、'\0'（空字符）等。

4. ASCII 和字符操作

  • char 类型常用于表示 ASCII 字符集，因为其范围足以表示 256 个字符。
  • 可以对 char 进行算术运算，例如：'A' + 1 的结果是字符 B。

5. 字符串与字符数组

  • char 数组可以表示 C 风格的字符串，例如 char str[] = "Hello";。

重复的子字符串

使用双倍字符串拼接的方式通常比基于质数的算法更快，因为双倍字符串方法直接利用了字符串的重复结构，避免了复杂的数学运算。下面详细对比两者的特点：

  1. 双倍字符串法

    • 时间复杂度：O(n)，其中 n 是字符串 s 的长度。
    • 原理：只需构建一个双倍字符串，并检查 s 是否出现在去头尾的双倍字符串中。
    • 优点：操作简单，代码短小，且在大多数常见情况下非常高效。
    • 缺点：适合一般情况，对于极端大字符串可能需要稍高的内存，但一般不会出现瓶颈。

  2. 质数方法

质数方法一般利用字符串的周期性，主要步骤如下：

1.	遍历 s 的每个可能周期长度 k，并尝试用 k 长度的子串拼接构成 s。
2.	为了优化，可以用质数判断 k 是否为潜在周期长度，借助字符串的长度 n 与 k 的整除关系。

•	时间复杂度：在最坏情况下，质数方法会有更高的时间复杂度，约为 O(n*sqrt(n))，因为需要循环判断每个可能的周期。
•	优点：基于质数的数学方法更具启发性，尤其适合学习和竞赛，但实现上稍复杂。
•	缺点：由于有循环判断，对于大字符串性能可能不如双倍字符串法。

结论

在实际应用中，双倍字符串方法通常比质数方法更快，因为它直接利用字符串的内存结构和查找操作，省去了周期性判断和质数检测的步骤。质数方法适合于更具挑战性的数学问题场景，但在解决字符串周期性问题时可能显得繁琐。

如果重点在于速度和简单实现，推荐使用双倍字符串法。

什么是字符串
字符串是若干字符组成的有限序列，也可以理解为是一个字符数组，但是很多语言对字符串做了特殊的规定，接下来我来说一说C/C++中的字符串。

在C语言中，把一个字符串存入一个数组时，也把结束符 ‘\0’存入数组，并以此作为该字符串是否结束的标志。

例如这段代码：

char a[5] = “asd”;
for (int i = 0; a[i] != ‘\0’; i++) {
}
在C++中，提供一个string类，string类会提供 size接口，可以用来判断string类字符串是否结束，就不用’\0’来判断是否结束。

例如这段代码:

string a = “asd”;
for (int i = 0; i < a.size(); i++) {
}
